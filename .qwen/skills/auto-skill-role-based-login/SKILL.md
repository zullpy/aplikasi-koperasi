## Overview
This skill implements two related behaviours for the Kopdes web application:

1. **Login redirect based on user role** – after a successful login the user is sent to the appropriate page:
   - `admin` → `selection-page/index.php`
   - `bendahara` or `purchase` → `transaksi-pembelian-food/index.php`
   - any other role → fallback to the selection page.
2. **Uniform authentication guard** – every public‑facing page now begins with a session check. If the user is not logged in they are redirected to the main login page (`../`).

Both changes are done by editing the relevant PHP files and inserting the same session‑validation snippet at the top of each page.

## Implementation Steps

1. **Update `process-login.php`**
   ```php
   // after setting $_SESSION['role']
   if ($_SESSION['role'] === 'admin') {
       header("Location: ../selection-page/index.php");
   } elseif (in_array($_SESSION['role'], ['bendahara', 'purchase'])) {
       header("Location: ../transaksi-pembelian-food/index.php");
   } else {
       header("Location: ../selection-page/index.php");
   }
   exit;
   ```
2. **Add authentication guard to each page** (e.g., `transaksi-pembelian-food/index.php`, `stok-barang-food/index.php`, `transaksi-penjualan-food/index.php`, `selection-page/index.php`, and all add‑cost pages):
   ```php
   session_start();
   if (!isset($_SESSION['id'])) {
       header('Location: ../');
       exit;
   }
   ```
   This snippet should be placed immediately after `<?php` and before any output.
3. **Consistency** – use the same relative redirect (`../`) for all pages so the guard works regardless of directory depth.
4. **Testing** – after deployment, attempt to access a protected page directly (e.g., `kbus.com/transaksi-pembelian-food`). The user should be redirected to the login page, then after successful login be sent to the page matching their role.
